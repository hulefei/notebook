定义UE4模块

July 14, 2023
UE4 定义一个module

在UE4中定义一个模块需要创建一个新的模块类。模块是游戏或应用的独立组件，可以包含代码，资源，或者两者。通常，一个模块都有自己的Build.cs文件，用于定义编译参数，以及一个包含模块的基本功能的模块类。

以下是一个基本模块类的示例：

cpp
#include "ModuleManager.h"  // FModuleManager用于注册你的模块

class FYourModuleName : public IModuleInterface
{
public:

  virtual void StartupModule() override
  {
    // 这个函数会在模块被加载的时候执行，这里可以执行你需要的初始化任务
  }

  virtual void ShutdownModule() override
  {
    // 这个函数会在模块被卸载的时候执行，这里可以执行你需要的清理任务
  }
};

IMPLEMENT_PRIMARY_GAME_MODULE( FYourModuleName, YourGameName, "YourGameName" );

这里"FYourModuleName"是你的模块名，"YourGameName"是你的游戏名。注意这两个名称要和你的项目名称相符。

这只是一个非常基础的模块类的示例。实际上你的模块类可能会包含更多的功能，比如处理模块的状态，导出模块的API，以及其他任何你需要的功能。

UE4 获取路径

在UE4中，可以使用FPaths类中的静态函数来获取各种路径。以下是一些常用的例子：

  1. 获取游戏的根目录：

    cpp
    FString BaseDir = FPaths::ProjectDir();

  2. 获取游戏的内容目录：

    cpp
    FString ContentDir = FPaths::ProjectContentDir();

  3. 获取游戏的插件目录：

    cpp
    FString PluginsDir = FPaths::ProjectPluginsDir();

  4. 获取游戏的保存目录：

    cpp
    FString SavedDir = FPaths::ProjectSavedDir();

  5. 获取游戏的配置目录：

    cpp
    FString ConfigDir = FPaths::ProjectConfigDir();

这些函数返回的路径是全路径，并以斜杠("/")结束。例如，"D:/MyGame/Content/"。

还有其他的函数可以获取和处理路径，比如FPaths::Combine()可以将多个路径组合成一个，FPaths::FileExists()可以检查一个文件是否存在。更多的详细信息可以参考官方文档或源代码中的注释。

注意，所有返回的路径都是标准化的，并且都使用"/"作为路径分隔符，这样就可以保证你的代码在所有平台上都能正常工作。

ProjectDir 是哪里，给我一个例子

ProjectDir() 是 FPaths 类的一个静态方法，返回的是当前 Unreal Engine 项目的根目录。例如，如果你的项目名为 "MyGame"，并且该项目存储在 "D:/UnrealProjects/MyGame/" 中，那么 FPaths::ProjectDir() 就会返回 "D:/UnrealProjects/MyGame/"。

注意，返回的路径以斜杠("/")结束。另外，路径字符串是完全解析过的，也就是说它是绝对路径，不包含任何 ".." 或 "." 元素。这样可以保证你的代码在所有平台上都能正常工作。

请确保在使用这个函数之前已经正确地启动了你的项目，因为在项目启动之前，UE4可能还不知道项目的根目录。

UE4 UENUM 转字符串

在UE4中，你可以使用GET_ENUMERATOR_NAME宏将UENUM值转换为字符串。请参阅以下示例：

首先，假设你有一个UENUM定义如下：

cpp
UENUM(BlueprintType)
enum class EYourEnum : uint8
{
    EnumValue1 UMETA(DisplayName = "EnumValue1"),
    EnumValue2 UMETA(DisplayName = "EnumValue2"),
    EnumValue3 UMETA(DisplayName = "EnumValue3")
};

然后，你可以使用以下代码将一个EYourEnum值转换为字符串：

cpp
EYourEnum YourEnumValue = EYourEnum::EnumValue1;
FString YourEnumValueAsString = GET_ENUMERATOR_NAME(TEXT("EYourEnum"), (int32)YourEnumValue).ToString();

这个GET_ENUMERATOR_NAME宏的第一个参数是你的枚举类型的名称，第二个参数是你的枚举值。然后你可以使用ToString方法将返回的FName对象转换为FString。

还有一种更直接的方式，如果你在一个UObject派生的类中使用这个枚举，你可以使用UEnum::GetValueAsString函数：

cpp
FString YourEnumValueAsString = UEnum::GetValueAsString(YourEnumValue);

这个函数需要你的枚举类型是一个UENUM(BlueprintType)，因为这样的枚举类型在运行时有反射信息，可以直接通过值获取到字符串。如果你的枚举类型不是BlueprintType，那么这个函数可能无法正常工作。

记住，这些方式获取的是枚举的名称，而不是枚举的DisplayName属性。如果你想获取DisplayName，你可能需要获取到UEnum对象，然后使用GetDisplayNameText函数。

获取displayname

如果你想获取UENUM的DisplayName，你可以使用以下的代码：

cpp
UENUM(BlueprintType)
enum class EYourEnum : uint8
{
    EnumValue1 UMETA(DisplayName = "Display EnumValue1"),
    EnumValue2 UMETA(DisplayName = "Display EnumValue2"),
    EnumValue3 UMETA(DisplayName = "Display EnumValue3")
};

// 获取EYourEnum的UEnum指针
UEnum* EnumPtr = FindObject<UEnum>(ANY_PACKAGE, TEXT("EYourEnum"), true);

if (!EnumPtr) return;  // 确保EnumPtr不是null

EYourEnum YourEnumValue = EYourEnum::EnumValue1;

// 使用GetDisplayNameText函数获取DisplayName
FText DisplayName = EnumPtr->GetDisplayNameTextByValue((int64)YourEnumValue);

这段代码首先使用FindObject获取EYourEnum类型的UEnum指针，然后通过这个指针调用GetDisplayNameTextByValue函数，传入枚举值的整数形式，就可以获取到DisplayName。

注意，GetDisplayNameTextByValue返回的是FText类型，这是Unreal Engine用于本地化文本的类型。你可以使用ToString或ToFString方法将它转换为FString。

cpp
FString DisplayNameString = DisplayName.ToString();

这样你就可以得到枚举的DisplayName字符串形式了。

UE4 接口转换

在Unreal Engine 4中，接口提供了一种方法来定义可以在不同类之间共享的功能，而无需依赖于类继承。通过使用接口，你可以在没有共同基类的情况下调用在各种对象上定义的同一功能。

UE4提供了一些用于处理接口的宏和函数。假设你有一个定义如下的接口：

cpp
UINTERFACE(BlueprintType)
class UYourInterface : public UInterface
{
    GENERATED_BODY()
};

class IYourInterface
{
    GENERATED_BODY()

public:
    // 你的接口函数定义
    UFUNCTION(BlueprintNativeEvent, BlueprintCallable, Category = "YourCategory")
    void YourFunction();
};

然后，如果你有一个UObject指针，你可以使用Cast<IYourInterface>来转换为接口，或者使用Implements<UYourInterface>来检查对象是否实现了接口：

cpp
UObject* YourObject = // 你的对象
if (YourObject->Implements<UYourInterface>())
{
    IYourInterface* YourInterface = Cast<IYourInterface>(YourObject);
    if (YourInterface)
    {
        YourInterface->YourFunction();
    }
}

这段代码首先检查YourObject是否实现了UYourInterface接口。如果实现了，就尝试将YourObject转换为IYourInterface类型的指针，并调用YourFunction函数。

注意，你只能将实现了接口的UObject派生的类的实例转换为接口。其他类型（比如原始C++类，或者没有UCLASS宏的类）不能被转换为接口。

另外，注意你的接口函数需要被标记为BlueprintNativeEvent或BlueprintImplementableEvent，这样它们才能在接口实现中被覆盖。否则，尽管你的对象实现了接口，但在运行时可能无法调用接口函数。

Ue4 ISkeletalMeshEditor 调用 GetPersonaToolkit

ISkeletalMeshEditor 是 UE4 的一个接口，它是用于处理骨骼网格编辑的功能的接口。在你的代码中，你可能在扩展编辑器功能或者编写插件时会用到这个接口。

要调用 ISkeletalMeshEditor 接口的 GetPersonaToolkit 方法，你首先需要有一个 ISkeletalMeshEditor 接口的实例。这通常在你的自定义编辑器插件或者扩展中，通过 UE4 的编辑器扩展点获取。

以下是一个调用 GetPersonaToolkit 的基本示例：

cpp
ISkeletalMeshEditor* SkeletalMeshEditor = // 你的ISkeletalMeshEditor实例
if(SkeletalMeshEditor)
{
    TSharedRef<IPersonaToolkit> PersonaToolkit = SkeletalMeshEditor->GetPersonaToolkit();
    // 在此处使用 PersonaToolkit
}

在这个例子中，首先我们检查 SkeletalMeshEditor 不是空指针，然后调用 GetPersonaToolkit 函数获取 IPersonaToolkit 接口的引用。

这个 IPersonaToolkit 接口提供了许多对于处理 Persona（UE4 的角色编辑器）相关的功能的方法。你可以在得到这个接口后，调用它的方法来获取或设置 Persona 相关的数据。

具体你要如何获取 ISkeletalMeshEditor 的实例，将取决于你的代码的上下文。例如，在一个自定义的编辑器插件或者窗口中，你可能会在创建或初始化的时候获得这个接口。

UE4 打印dialog

在Unreal Engine 4中，有多种方式可以打印或显示消息给用户。以下是一些最常见的方式：

  1. 打印到屏幕：UE4提供了GEngine->AddOnScreenDebugMessage函数，可以将消息直接打印到游戏屏幕上。这个函数在快速调试时非常有用：

    cpp
    GEngine->AddOnScreenDebugMessage(-1, 5.f, FColor::Red, TEXT("This is your message."));

    这行代码会将字符串"This is your message."以红色字体打印到屏幕上，持续时间为5秒。

  2. 打印到控制台：如果你想将消息打印到控制台或日志文件，可以使用UE_LOG宏：

    cpp
    UE_LOG(LogTemp, Warning, TEXT("This is your message."));

    这行代码会将字符串"This is your message."打印到控制台和日志文件，使用的是LogTemp类别，级别为Warning。

  3. 显示对话框：如果你想显示一个对话框给用户，可以使用FMessageDialog::Open函数。这个函数可以在编辑器环境中打开一个模态对话框，显示你的消息：

    cpp
    FText DialogTitle = FText::FromString("Dialog Title");
FText DialogContent = FText::FromString("This is your message.");
FMessageDialog::Open(EAppMsgType::Ok, DialogContent, &DialogTitle);

    这段代码会打开一个有标题和内容的对话框，只有一个OK按钮。

请注意，对话框通常只在编辑器环境中使用，如果你在运行的游戏中使用，可能不会看到任何效果。在运行的游戏中显示消息，通常会使用HUD或UMG UI系统来创建自定义的界面元素。

以上方法中的所有字符串都使用TEXT宏来创建。这个宏确保了字符串是以UTF-16编码的，这样就可以在所有平台上正确地显示所有语言。

const 指针 转 非const指针

在C++中，转换const指针到非const指针是可能的，但不推荐这样做，因为这会破坏const的安全性。const关键字是用来保证一个对象在生命周期中的不可变性，如果你尝试去移除const，那么可能会导致未定义行为。

但如果你确实需要这样做，你可以使用const_cast来实现。这是一个C++的类型转换运算符，可以用来修改类型的const或volatile属性。以下是一个例子：

cpp
const int* const_ptr = new int(10);
int* non_const_ptr = const_cast<int*>(const_ptr);
*non_const_ptr = 20;  // 现在你可以修改non_const_ptr指向的值

这段代码首先定义了一个指向const int的指针const_ptr，然后使用const_cast将其转换为一个指向int的非const指针。这样你就可以修改non_const_ptr指向的值了。

然而，这种操作应当谨慎使用。如果一个对象被声明为const，那么应当尽量不去修改它的值。如果你需要修改const对象的值，那么可能是你的设计存在问题。在许多情况下，更好的解决方案是使用mutable关键字，或者重新设计你的类和函数，以避免需要修改const对象。

UE asset

在 Unreal Engine 中，"asset" 是一个非常重要的概念。Asset 是游戏中使用的所有内容的基本单位，包括模型、材质、纹理、声音、蓝图、脚本、关卡等。每个 asset 都是一个独立的文件，它可以在编辑器中创建、修改和保存。

每种类型的 asset 都有一个相应的类与之关联，比如 UStaticMesh 代表一个静态网格模型，UMaterial 代表一个材质，USoundWave 代表一个声音文件等。这些类都继承自 UObject，这是 Unreal Engine 中所有 asset 类的基类。

你可以在内容浏览器中查看、导入、创建和组织你的 assets。每个 asset 都有一个唯一的路径，比如 "/Game/MyFolder/MyAsset.MyAsset"，你可以通过这个路径在代码中加载和使用 asset。以下是加载一个模型的例子：

cpp
UStaticMesh* MyMesh = Cast<UStaticMesh>(StaticLoadObject(UStaticMesh::StaticClass(), nullptr, TEXT("/Game/MyFolder/MyMesh.MyMesh")));

你也可以使用 Unreal Engine 的其他系统来管理 assets，比如资产管理器（Asset Manager）可以用来处理异步加载和卸载 assets，资产捆绑（Asset Bundles）可以用来标记哪些 assets 需要一起打包，数据驱动框架（Data Driven Framework）可以用来在数据表中定义 assets 的属性等。

总的来说，Unreal Engine 中的 asset 是实现游戏内容的关键，理解和掌握如何使用它们对于 Unreal Engine 开发者来说非常重要。

ue4 assert

在 Unreal Engine 4（UE4）中，断言是一种用于在代码中检查条件并在条件不满足时发出警告的编程技术。这在开发过程中特别有用，因为它可以帮助我们在运行时检测并捕获潜在的错误。

在 UE4 中，我们可以使用以下几种断言函数：

  1. check(): 这个断言函数在表达式为假时会立即终止程序。在编译发行版本时，check() 断言不会被移除，所以你只应当在确定错误会导致程序无法继续运行的情况下使用它。

    cpp
    check(Pointer != nullptr);

  2. ensure(): 与 check() 不同，ensure() 断言在表达式为假时不会终止程序，而是会打印一条错误信息并继续执行。如果同一个 ensure() 断言在同一次函数调用中被多次触发，它只会打印一次错误信息。这可以用于检测那些不一定会导致程序崩溃的错误。

    cpp
    ensure(Variable != 0);

  3. checkSlow() 和 ensureSlow(): 这两个断言函数的功能与 check() 和 ensure() 类似，但它们只在编译调试版本和开发版本时有效，在编译发行版本时会被移除。它们用于检测那些可能会影响性能的错误。

    cpp
    checkSlow(Variable > 0);
ensureSlow(Pointer != nullptr);

以上所有断言函数都可以接受一个表达式作为参数。如果这个表达式为假，断言就会被触发。断言函数还会在控制台和日志中打印出被触发的断言的源文件和行号，以帮助你更快地定位错误。

ue4 show dialog

在 Unreal Engine 4 中，如果你想在编辑器中显示对话框，你可以使用 FMessageDialog 类。它提供了一种简单的方式来显示模态对话框，并获取用户的输入。下面是一个示例：

cpp
FText DialogTitle = FText::FromString("Dialog Title");
FText DialogText = FText::FromString("This is a dialog message.");
EAppReturnType::Type DialogResult = FMessageDialog::Open(EAppMsgType::YesNo, DialogText, &DialogTitle);

if(DialogResult == EAppReturnType::Yes)
{
    // User clicked Yes
}
else if(DialogResult == EAppReturnType::No)
{
    // User clicked No
}

在这个示例中，FMessageDialog::Open 方法会打开一个带有标题和消息的对话框，对话框有 "Yes" 和 "No" 两个按钮。用户点击按钮后，方法会返回一个枚举值，表示用户点击的按钮。

然而，请注意，这只在编辑器中有效。如果你想在游戏运行时显示对话框，你需要使用 UMG（Unreal Motion Graphics）来创建自定义的 UI 元素。在UMG中，你可以创建自定义的面板，添加按钮，文本，滑动条等元素，然后在用户与这些元素互动时处理相应的事件。

UMG 提供了更加灵活和强大的 UI 创建工具，但也需要更多的工作。如果你只是需要一个简单的对话框，并且只在编辑器中需要它，那么 FMessageDialog 应该就足够了。
